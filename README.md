# 1sObjectManager
НаполнительОбк - предназначен для создания элементов справочника или создание документов на основе табличных данных

Как Заполнить ТабДок:
В заголовок пишем наименование Реквизита.
Если реквизит ссылочного типа Тогда после наименования реквизита пишем разделитель "_ref_"
и имя ссылочного типа (например Организация_ref_Организации)

Разделитель "_ref_" - позволяет заполнять ссылочным знч, поиск знч реализован по "Коду", 
(а также Если код нам не известен Тогда можно ввести наименование и поиск отработает по нему)
Просто надо учитывать что код - это "уникальное" знч, а наименование может повторятся у разных элементов!

Когда используется разделитель ("_ref_") то строка разбивается на два элемента:
мСоставПоля[0] - имя реквизита
мСоставПоля[1] - имя ссылочного типа

Пример заголовка:
Если необходимо заполнить реквизит "Родитель" значением ссылочного типа СправочникСсылка "Номенклатура"
Тогда заголовок колонки должен быть таким: Родитель_ref_Номенклатура
Знч данной колонки должен содержать "Код" конкретного элемента справочника "Номенклатура"

Если необходимо заполнить реквизит "Наименование" примитивным типом (например строкой) Тогда заголовок колонки будет таким:
например "ФИО", а значение данной колонки будет просто строка "Какое либо наименование"

ВНИМАНИЕ: 

ЭТО ОБЯЗАТЕЛЬНАЯ КОЛОНКА если необходимо создавать какие либо данные на основании выходных данных!
 
Добавьте колонку с заголовком "НумерацияСвязи" которую необходимо заполнить номером по порядку!
(по этому полю мы будем ориентироваться (в таблице результат) с каким кодом был создан новый элемент
просто нумеруем строки которые собираемся обрабатывать(загружать в справочник) - это и будет колонка НумерацияСвязи)

НумерацияСвязи - зарезервированное обработкой имя колонки, для связи входных и выходных данных.


Для использования выходных данных в наполнении других объектов:
1. Копируйте выходные данные в excel
2. Выполните сортировку по полю "НумерацияСвязи"
